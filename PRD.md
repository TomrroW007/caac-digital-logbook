# CAAC 数字飞行记录本 - 产品需求文档 (PRD)

**当前状态:** Stable / Production-Ready  
**当前版本:** V1.5.0  

## 1. 产品概述
CAAC 数字飞行记录本旨在为中国民航飞行员提供一个安全、合规、离线可用的日志管理工具，彻底替代传统的纸质记录本。本产品严格遵循 CAAC 审计标准，支持飞行与模拟机记录隔离计算、复杂时区处理以及无缝的多端同步。

## 2. 核心里程碑与实施状态

### ✅ Phase 1-6: 基础建设与离线业务闭环 (已完成)
* **核心 UI 框架构建：** React Navigation 原生导航。
* **离线数据库集成：** WatermelonDB 模型建立 (`LogbookRecord`)。
* **表单与校验引擎：** 引入 `ComplianceValidator.ts`，严格控制飞行时间逻辑（如：安全机长、带飞时间不得超过轮挡时间）。
* **90 天近期经历：** 使用北京时间 (UTC+8) 基准计算 90 天内昼/夜起落次数。

### ✅ Phase 7: 云端数据同步与导入 (已完成)
* **Supabase 后端集成：** 引入免费版 Supabase 替代复杂的自建后端。
* **双向同步机制：** 基于 WatermelonDB `sync()` 实现增量拉取/推送。
* **历史数据导入：** `ImportService` 支持批量解析用户历史 Excel，并具备去重（指纹）机制。

### ✅ Phase 8: 审计级导出与数据重构 (已完成)
* **类型分流导出：** 将“真实飞行”与“模拟机”记录在 PDF 与 Excel 导出中彻底分离。
* **防漂移时区锁：** 在导出界面禁用本地动态时区，强制要求用户选择 UTC 或 北京时间 (LT)，满足 CCAR-121 疲劳审计连续性要求。
* **PDF 排版引擎：** 实现 HTML/DOM 分页引擎，确保表格头部渲染正确，并增加局方强制要求的底部教员/审查员签字栏。

### ✅ Phase 9: 鉴权稳定性与 UX 深度打磨 (已完成 - V1.5.0)
* **无密码安全登录 (Passwordless OTP)：** 弃用容易遗忘的传统密码，改用 6 位邮箱一次性验证码，提升飞行员登录体验。
* **多端容灾与防杀后台：** 增加 `AsyncStorage` 缓存 OTP 状态，用户切后台查看邮箱验证码返回时，页面不刷新、不丢失状态。
* **限流防刷机制：** 增加 60 秒验证码重发倒计时，保护云端接口资源配额。
* **多账号安全拦截：** 检测本地已有数据与新登录账号不匹配时，弹出“数据污染警告”，指导用户清理本地缓存。

## 3. 数据模型 (Schema V4)
核心业务表：`logbook_records`
* 支持离线 UUID 标识。
* 服务端触发器 `moddatetime` 强制接管 `updated_at`，彻底杜绝客户端设备时钟不准导致的同步拉取遗漏。
* 数据隔离：Supabase 层面执行 Row Level Security，所有查询隐式附带 `user_id = auth.uid()`。
